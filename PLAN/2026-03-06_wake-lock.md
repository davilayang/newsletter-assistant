# Wake Lock API for LiveKit Voice Sessions

## Context

On mobile browsers (especially Android/DuckDuckGo), the screen turning off suspends WebSocket connections, killing active LiveKit voice sessions. Adding Wake Lock API support keeps the screen on during a voice session, preventing disconnection. A silent audio fallback handles browsers that don't support the Wake Lock API.

## Changes

**File:** `src/frontend/livekit_widget.py`

1. Add `_wakeLock = null;` global variable alongside `_room` and `_micTrack`
2. In `lkConnect()` — acquire wake lock after successful connection (after mic track is published)
3. In `lkDisconnect()` — release wake lock before disconnecting the room
4. In `RoomEvent.Disconnected` listener — release wake lock as a safety net (handles unexpected disconnects)
5. Add silent audio fallback for browsers that don't support Wake Lock API (e.g. DuckDuckGo Android) — play a tiny silent audio loop to prevent the browser from suspending the tab

No other files need changes.
