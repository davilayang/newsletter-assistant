# Frontend — Plan

A browser-based UI that wraps the voice agent session, shows live transcription,
and surfaces the newsletter content and notes without needing the terminal.

---

## Framework Options

### Option A — Streamlit *(Recommended for quick start)*

Pure Python, minimal boilerplate. Use `st.components.v1.html` to embed a small
LiveKit JS snippet for the actual audio, then build the rest of the page in Python.

**Pros:** Fastest to ship, zero JS knowledge needed, great for dashboards.
**Cons:** Limited real-time reactivity (polling needed for live transcript updates);
           audio widget has to be a JS embed, not native Streamlit.

### Option B — Gradio

`gr.ChatInterface` gives a chat window out of the box. Custom JS hooks let you
bridge to LiveKit audio.

**Pros:** ChatInterface component is very close to what we need; easy streaming.
**Cons:** Less flexible for a multi-panel layout (articles sidebar, notes panel).

### Option C — Reflex

Full Python stack — compiles to React under the hood. Native real-time state
updates via websocket, no polling. More setup but best long-term.

**Pros:** Real Python reactivity, proper component library, no JS needed.
**Cons:** Higher initial complexity; relatively new framework.

### Option D — FastHTML + HTMX

Lightweight HTML-over-the-wire framework. Very small footprint.

**Pros:** Simple server-side rendering, HTMX handles live updates cheaply.
**Cons:** Less visual polish than Reflex/Streamlit without more work.

---

## LiveKit Audio — Integration Strategy

The voice pipeline (mic → STT → LLM → TTS) runs entirely in the browser via
WebRTC. Python frameworks cannot handle raw audio directly. Options:

**1. Embed LiveKit's JS client SDK** (recommended)
- Backend: FastAPI endpoint generates a LiveKit room token (Python LiveKit SDK).
- Frontend: Small `<script>` block (or `st.components.v1.html` / Gradio `HTML`
  component) connects to the room using `livekit-client` JS.
- Keeps the Python UI as the shell; audio is a self-contained JS widget.

**2. Link to LiveKit Playground**
- LiveKit hosts a generic web client at `agents-playground.livekit.io`.
- Point it at your LiveKit Cloud URL — zero frontend code for audio.
- Good for early testing; less polished for production.

**3. Separate React audio component**
- A minimal React app (`create-livekit-app`) handles only audio/video.
- Embedded inside the Python app via iframe.

## Features

### Core (Phase 1)

| Feature | Description |
|---|---|
| **Chat transcription** | Live scrolling conversation log (user turns + agent turns). Timestamps on each turn. |
| **Clickable URLs in transcript** | Detect `https://...` in agent responses, render as `<a href>` links that open in a new tab. |
| **Article panel** | Today's newsletter article list (titles + URLs). Clicking a URL opens the article. Populated by calling `get_todays_newsletter` or reading `raw_store` directly. |
| **Voice controls** | Mute toggle, End session button, mic level indicator. |
| **Notes panel** | Live view of `NOTES/<date>_medium-notes.md`. Auto-refreshes when a note is saved. |

### Nice-to-Have (Phase 2)

| Feature | Description |
|---|---|
| **Knowledge search** | Text input → calls `vector_store.search()` directly, shows results without going through the voice agent. Useful for quick lookups. |
| **Article reading mode** | Click an article in the list → renders its cached markdown from `raw_store` in a readable panel. Avoids re-fetching. |
| **Pipeline status badge** | Shows last pipeline run time and article count. Pulls from `raw_store.get_all_articles()`. |
| **Session history** | Browse past conversation transcripts saved to `NOTES/`. |
| **Dark mode** | Stylesheet toggle — useful for a morning reading session before the room lights up. |
| **TTS audio player** | Optionally replay the last agent utterance (useful if you missed something). |

## Backend Additions Required

### Token endpoint (new)

`GET /token?room=newsletter-{date}` — generates a LiveKit participant JWT.
Minimal FastAPI app, can live in `src/frontend/server.py`.

```python
from livekit.api import AccessToken, VideoGrants

@app.get("/token")
def get_token():
    token = (
        AccessToken(api_key, api_secret)
        .with_grants(VideoGrants(room_join=True, room="newsletter"))
        .with_identity("user")
        .to_jwt()
    )
    return {"token": token, "url": settings.livekit_url}
```

### Transcript store (new or reuse NOTES/)

Options:
- **File-based:** Agent writes turns to `NOTES/<date>_session-transcript.json`;
  frontend polls the file every second (simplest).
- **In-memory via SSE:** Agent pushes events over a Server-Sent Events endpoint;
  frontend subscribes. Real-time but slightly more complex.
- **SQLite:** Add a `transcript` table to `articles.db`. Most robust.

The file-based approach is consistent with the existing `NOTES/` pattern and
requires no new dependencies.

## Implementation Steps

### Step 1 — Minimal Streamlit shell (~2 hours)

1. `uv add streamlit livekit-api fastapi uvicorn` (FastAPI for token endpoint)
2. Create `src/frontend/server.py` — FastAPI app with `/token` endpoint
3. Create `src/frontend/app.py` — Streamlit app:
   - Left column: article list (reads `newsletters.yaml` + calls `raw_store`)
   - Right column: chat transcript (polls `NOTES/<date>_transcript.json`)
   - Bottom: notes panel (reads today's notes file)
   - Embed a JS snippet in `st.components.v1.html` for LiveKit audio
4. Add poe task: `uv run poe frontend`

### Step 2 — Transcript persistence (~1 hour)

1. In `src/agent/agent.py` attach `session.on("conversation_item_added")`:
   ```python
   def _on_item(item):
       # append {role, text, ts} to NOTES/<date>_transcript.json
   session.on("conversation_item_added", _on_item)
   ```
2. Streamlit polls the file with `st.empty()` + `time.sleep(1)` loop.

### Step 3 — URL linkification in transcript (~30 min)

Simple regex pass over transcript text before rendering:
```python
import re
URL_RE = re.compile(r'(https?://\S+)')
def linkify(text):
    return URL_RE.sub(r'<a href="\1" target="_blank">\1</a>', text)
```
Render with `st.markdown(linkify(text), unsafe_allow_html=True)`.

### Step 4 — Knowledge search panel (~1 hour)

Add a sidebar text input that calls `vector_store.search(query)` directly and
shows results with titles, authors, and snippet previews.
